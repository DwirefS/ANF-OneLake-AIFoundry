1. Validate Azure Subscription Requirements

Goal: Ensure your Azure subscription is ready to run the lab without permission or provider issues.

  1. Login — Open https://portal.azure.com and sign in with your Azure work account.
  2. Verify Subscription Role
    • Search for Subscriptions in the portal.
    • Select the subscription you will use for the lab.
    • Go to Access control (IAM).
    • Confirm your user has Owner or Contributor + User Access Administrator.
  3. Register Resource Providers
    • On the Subscriptions blade, under Settings, select Resource providers.
    • Confirm the following are registered:
      • Microsoft.NetApp
      • Microsoft.Search
      • Microsoft.CognitiveServices
      • Microsoft.MachineLearningServices
      • Microsoft.Fabric
    • If any show Not registered, select the provider and click Register.

2. Configure Azure NetApp Files

Goal: Create an Azure NetApp Files volume and enable S3-compatible (Object REST API) access.

Preview Requirement: The Object REST API for Azure NetApp Files is in Public Preview. Ensure your subscription is whitelisted before proceeding.

2.1 Create NetApp Account and Capacity Pool

  1. In the Azure portal, search for Azure NetApp Files.
  2. Click + Create.
    • Name: Workshop-NetApp-Account
    • Resource group: Your resource group
    • Region: Choose a supported region (e.g., East US)
  3. Click Create.
  4. Open the NetApp account, select Capacity pools, then click + Add pool.
    • Pool name: Workshop-Pool
    • Service level: Standard
    • Size: 1 TiB (minimum)
  5. Click Create.

2.2 Create a Volume

  1. Select Volumes, then click + Add volume.
    • Name: anf-finance-vol
    • Capacity pool: Select the pool from 2.1
    • Quota: 100 GiB
    • Virtual network & delegated subnet: Select your VNet
    • Protocol: NFS (NFSv3 recommended)
  2. Click Review + Create, then Create.

2.3 Enable Object (S3-Compatible) Access

  1. Generate a certificate: 
    openssl req -x509 -newkey rsa:4096 -keyout private.key -out cert.pem -days 365 -nodes
  2. Create a Bucket:
    • In the volume blade, under Storage service, click Buckets.
    • Click + Create or update bucket.
    • Name: finance-data
    • Path: /
    • Permissions: Read and Write
    • Upload the cert.pem file.
  3. Capture Credentials: Copy and save the Endpoint URL, Access Key, and Secret Key.

2.4 Download Lab Data

Download the test_data folder from this repository to your local machine.

2.5 Upload Lab Data

  1. Install a client such as Cyberduck or S3 Browser.
  2. Create a new S3 connection using your ANF endpoint, access key, and secret key. Enable path-style addressing if prompted.
  3. Upload the invoices/ and financial_statements/ folders to the bucket root.

3. Create a OneLake Shortcut to Azure NetApp Files

Goal: Virtualize Azure NetApp Files data into Microsoft Fabric OneLake without copying it.

3.1 Deploy On-Premises Data Gateway

Note: This gateway is required by Microsoft Fabric when accessing private or VNet-isolated S3-compatible endpoints. Azure NetApp Files itself does not require a gateway.
  1. Create a Windows VM in the same VNet as your ANF volume (Standard_D2s_v3 recommended).
  2. RDP into the VM.
  3. Download and install the On-premises data gateway (Standard mode).
  4. Sign in and register a new gateway named ANF-Gateway.
  5. Verify the gateway shows Online.

3.2 Create Fabric Workspace

  1. Go to app.fabric.microsoft.com.
  2. Select Workspaces → + New workspace.
    • Name: Financial_RAG_Workshop
    • License mode: Trial or Fabric capacity
  3. Click Create.

3.3 Create Connection and OneLake Shortcut

  1. In Fabric, open Settings → Manage connections and gateways.
  2. Select + New connection → Amazon S3 Compatible.
    • Gateway: ANF-Gateway
    • Server: ANF endpoint IP
    • Authentication: Access key / Secret key
  3. Create a Lakehouse:
    • Workspace → + New item → Lakehouse
    • Name: FinDataLake
  4. Create the Shortcut:
    • Open Lakehouse, right-click Files, click New shortcut.
    • Source: Amazon S3 Compatible
    • Connection: Select the ANF connection
    • Bucket: finance-data
    • Name: anf_shortcut
  5. Verify folders are visible under the shortcut.

4. Index OneLake Data Using Azure AI Search

Goal: Make OneLake data searchable and usable for RAG.

4.1 Create Azure AI Search Service

  1. In the Azure portal → Create a resource → Azure AI Search.
  2. Select Basic tier (required for Semantic Search) or Standard.
  3. After creation, go to Identity → Enable System-assigned managed identity.

4.2 Assign Permissions

  1. Open your Fabric workspace (Financial_RAG_Workshop).
  2. Select Manage access.
  3. Add the Azure AI Search Managed Identity (search by service name) as a Member or Contributor.
Workshop Guardrails:
  • Avoid Microsoft Purview sensitivity labels on the Lakehouse or files.
  • Use reasonably sized text-based documents (PDF, TXT, CSV).

4.3 Import and Vectorize Data

  1. Open Azure AI Search → Import and vectorize data.
  2. Source: OneLake (Microsoft Fabric).
  3. Connection: Select your workspace (Financial_RAG_Workshop), Lakehouse (FinDataLake), Path: Files/anf_shortcut.
  4. Vectorization: Azure OpenAI — select your embedding model.
  5. Index Settings: Content: Searchable, Metadata_storage_path: Retrievable (must be checked).
  6. Create and run the indexer.

5. Connect OneLake to Azure AI Foundry

Goal: Enable Foundry to ground agents on indexed enterprise data.

5.1 Create Azure AI Services Resource

  1. Azure Portal → Create a resource → Azure AI services (Multi-service account).
    • Name: Workshop-AI-Services
    • Region: East US 2 (or Sweden Central, West US 3)
    • Pricing Tier: Standard S0
  2. Deploy Models:
    • Go to Model deployments → Manage Deployments.
    • Deploy gpt-4o (name it gpt-4o).
    • Deploy text-embedding-3-small (name it text-embedding-3-small).

5.2 Assign User Permissions

  1. Go to Workshop-AI-Services → Access control (IAM).
  2. Add role assignment:
    • Role: Cognitive Services OpenAI User
    • Members: Select your user account
  3. Click Review + assign.

5.3 Create Foundry Project

  1. Go to ai.azure.com.
  2. Click + Create Project.
    • Name: Finance-RAG-Project
    • Hub: Create new hub named Finance-Hub
    • Resource Group: Use your existing one
  3. Connect Resources:
    • Azure OpenAI: Select Workshop-AI-Services
    • Azure AI Search: Select the service from Step 4
  4. Click Create.

6. Run and Validate a Grounded Agent

Goal: Confirm the agent answers questions using ANF-hosted data.

6.1 Configure Data Source

  1. In your Project → Left Menu → Data (or "Indexes").
  2. + New connection → Azure AI Search.
  3. Select the Index created in Step 4.

6.2 Create & Test Agent

  1. Go to Agents, create a new agent.
    • Model: Select the gpt-4o deployment.
    • Knowledge: Select the Search Index connection.
  2. System Prompt: 
    You are a Financial Auditor.
Use the attached financial data to answer questions.
Always cite the document name.
If data is in a CSV, calculate totals by summing rows.
  3. Test with example queries:
    • "What is the total spend for vendor 'OfficeMax'?"
    • "Show me all transactions from Q1 2025."

7. Troubleshooting

  • Gateway Error: Ensure the gateway VM has outbound 443 access to the ANF IP.
  • OneLake Shortcut Empty: Verify the bucket name in the shortcut settings matches the ANF bucket exactly.
  • Indexer Error (403): Re-check Step 4.2. The Search Service Managed Identity must have permission on the Fabric workspace.
  • Agent "I don't know": Check "Strictness" (set to 3) and ensure the Indexer ran successfully (document count > 0).
  • Permissions Error in Foundry: Ensure you assigned yourself Cognitive Services OpenAI User in Step 5.2.
