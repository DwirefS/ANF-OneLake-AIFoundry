Architecture

The Zero-Copy AI Data Path

Data stays in its authoritative location throughout the entire pipeline. No ETL, no duplication, no re-platforming.

Layer 1 · Storage

Azure NetApp Files

Enterprise NFS/SMB volume with S3-compatible Object REST API. Your data stays here.

Layer 2 · Integration

OneLake Shortcut

Microsoft Fabric virtualizes access via S3-compatible shortcuts through a Data Gateway. Zero copies.

Layer 3 · Indexing

Azure AI Search

OneLake indexer extracts, chunks, and vectorizes content for hybrid retrieval.

Layer 4 · Intelligence

AI Foundry Agents

GPT-4o agents grounded on enterprise data. Ask questions, get cited answers.

Why Zero-Copy Matters

Enterprise data stays governed in Azure NetApp Files. No new storage silos. No brittle ingestion pipelines. AI agents operate directly on trusted data through virtualization.

Prerequisites

Ensure your environment meets these requirements before starting the workshop.

✓
Azure subscription with Owner or Contributor role
✓
Object REST API preview enabled on subscription
✓
Microsoft Fabric capacity (Trial or paid)
✓
Resource providers registered (NetApp, Search, etc.)
✓
S3 client tool (Cyberduck or S3 Browser)
✓
Access to a region with GPT-4o availability
