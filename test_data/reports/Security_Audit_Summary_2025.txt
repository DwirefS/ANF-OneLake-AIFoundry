ANNUAL SECURITY AUDIT SUMMARY — 2025
Prepared by: Michael Torres, CISO
Audit Firm: Deloitte Cybersecurity Practice
Audit Period: January 1, 2025 — December 31, 2025
Report Date: January 15, 2026

1. AUDIT SCOPE
This audit evaluated Contoso Financial Services' information security controls, data protection practices, and compliance with SOC 2 Type II, PCI DSS v4.0, and internal security policies. The audit covered all production systems, including Azure cloud infrastructure, Azure NetApp Files storage, Microsoft Fabric analytics platform, and Azure AI services.

2. OVERALL RATING: SATISFACTORY (with 3 findings)

3. KEY FINDINGS

Finding 1: MODERATE — S3 Access Key Rotation Not Automated
Description: Azure NetApp Files Object REST API access keys are generated with manual lifespans (365 days). There is no automated rotation mechanism.
Risk: If a key is compromised, it could provide unauthorized access to financial documents for up to 365 days.
Recommendation: Implement automated key rotation every 90 days using Azure Automation or Logic Apps. Store keys in Azure Key Vault with expiration alerts.
Management Response: Accepted. Will implement automated rotation by March 2026. James Park (Engineering) assigned as owner.

Finding 2: LOW — On-Premises Data Gateway (OPDG) Running on Single VM
Description: The OPDG connecting Azure NetApp Files to Microsoft Fabric OneLake runs on a single Windows VM (ANF-Gateway-VM). No failover or redundancy exists.
Risk: Gateway VM failure would disrupt OneLake data virtualization and AI Search indexing until the VM is recovered.
Recommendation: Deploy a second OPDG VM in a different availability zone and configure the gateway cluster for high availability.
Management Response: Accepted. Redundant gateway planned for Q2 2026 deployment. Estimated cost: $140/month for second B2s VM.

Finding 3: LOW — AI Agent Interaction Logs Lack Detailed Classification
Description: Azure AI Foundry agent interactions are logged, but logs do not classify the sensitivity level of data accessed during each query.
Risk: Compliance auditors may not be able to determine if RESTRICTED data was accessed without manual log review.
Recommendation: Implement structured logging that tags each AI agent response with the data classification level of source documents used for grounding.
Management Response: Accepted. Will add classification tagging to the RAG pipeline metadata by April 2026.
